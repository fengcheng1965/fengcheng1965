## 🎯 后台系统开发计划

### 第一阶段：后端API开发（预计2-3周）

#### 1. 技术栈选择

**后端框架**：Node.js + Express.js  
**数据库**：MySQL 8.0  
**ORM**：Sequelize  
**身份验证**：JWT（JSON Web Token）  
**API文档**：Swagger/OpenAPI  

#### 2. 数据库设计

**用户表（users）**
```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    email VARCHAR(100) UNIQUE,
    role ENUM('admin', 'manager', 'staff') DEFAULT 'staff',
    status ENUM('active', 'inactive') DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

**客户表（customers）**
```sql
CREATE TABLE customers (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    phone VARCHAR(20) NOT NULL,
    email VARCHAR(100),
    company VARCHAR(100),
    address TEXT,
    status ENUM('potential', 'active', 'inactive') DEFAULT 'potential',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_phone (phone),
    INDEX idx_status (status)
);
```

**留言表（messages）**
```sql
CREATE TABLE messages (
    id INT PRIMARY KEY AUTO_INCREMENT,
    customer_id INT,
    name VARCHAR(100) NOT NULL,
    phone VARCHAR(20) NOT NULL,
    email VARCHAR(100),
    subject VARCHAR(200),
    content TEXT NOT NULL,
    status ENUM('pending', 'processing', 'replied', 'closed') DEFAULT 'pending',
    reply_content TEXT,
    replied_by INT,
    replied_at TIMESTAMP NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (customer_id) REFERENCES customers(id),
    FOREIGN KEY (replied_by) REFERENCES users(id),
    INDEX idx_status (status),
    INDEX idx_created_at (created_at)
);
```

**产品表（products）**
```sql
CREATE TABLE products (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(200) NOT NULL,
    category VARCHAR(50),
    price DECIMAL(10, 2) NOT NULL,
    original_price DECIMAL(10, 2),
    stock INT DEFAULT 0,
    description TEXT,
    specifications TEXT,
    image_url VARCHAR(500),
    status ENUM('active', 'inactive') DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_category (category),
    INDEX idx_status (status)
);
```

**订单表（orders）**
```sql
CREATE TABLE orders (
    id INT PRIMARY KEY AUTO_INCREMENT,
    order_no VARCHAR(50) UNIQUE NOT NULL,
    customer_id INT,
    total_amount DECIMAL(10, 2) NOT NULL,
    status ENUM('pending', 'paid', 'shipped', 'completed', 'cancelled') DEFAULT 'pending',
    payment_method VARCHAR(50),
    shipping_address TEXT,
    contact_name VARCHAR(100),
    contact_phone VARCHAR(20),
    remark TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (customer_id) REFERENCES customers(id),
    INDEX idx_order_no (order_no),
    INDEX idx_status (status),
    INDEX idx_created_at (created_at)
);
```

#### 3. API接口设计

**认证接口**
```
POST /api/auth/login          # 登录
POST /api/auth/register       # 注册（管理员）
POST /api/auth/refresh        # 刷新Token
POST /api/auth/logout         # 退出登录
```

**客户管理接口**
```
GET    /api/customers         # 获取客户列表
GET    /api/customers/:id     # 获取客户详情
POST   /api/customers         # 创建客户
PUT    /api/customers/:id     # 更新客户
DELETE /api/customers/:id     # 删除客户
GET    /api/customers/search  # 搜索客户
```

**留言管理接口**
```
GET    /api/messages          # 获取留言列表
GET    /api/messages/:id      # 获取留言详情
PUT    /api/messages/:id/reply # 回复留言
PUT    /api/messages/:id/status # 更新留言状态
DELETE /api/messages/:id      # 删除留言
GET    /api/messages/stats    # 留言统计
```

**产品管理接口**
```
GET    /api/products          # 获取产品列表
GET    /api/products/:id      # 获取产品详情
POST   /api/products          # 创建产品
PUT    /api/products/:id      # 更新产品
DELETE /api/products/:id      # 删除产品
```

**订单管理接口**
```
GET    /api/orders            # 获取订单列表
GET    /api/orders/:id        # 获取订单详情
PUT    /api/orders/:id/status # 更新订单状态
GET    /api/orders/stats      # 订单统计
```

## 🔐 管理员权限说明

### 用户角色定义

| 角色 | 权限级别 | 可访问功能 | 操作限制 |
|------|----------|------------|----------|
| **超级管理员** | 最高 | 所有功能 | 无限制 |
| **管理员** | 高 | 大部分功能 | 不能管理用户 |
| **客服人员** | 中 | 客户管理、留言管理 | 不能删除数据 |
| **销售人员** | 低 | 客户查看、订单查看 | 只能查看自己的数据 |

### 权限详细说明

#### 1. 超级管理员（admin）

**权限范围**：
- ✅ 所有系统功能
- ✅ 用户管理（增删改查）
- ✅ 权限设置
- ✅ 数据备份与恢复
- ✅ 系统日志查看

**访问路径**：
- 登录页面：`/admin/login`
- 仪表盘：`/admin/dashboard`
- 所有管理功能模块

#### 2. 管理员（manager）

**权限范围**：
- ✅ 客户管理（增删改查）
- ✅ 留言管理（查看、回复、删除）
- ✅ 产品管理（增删改查）
- ✅ 订单管理（查看、更新状态）
- ❌ 用户管理（不能管理其他用户）
- ❌ 系统设置（不能修改系统配置）

**访问路径**：
- 登录页面：`/admin/login`
- 仪表盘：`/admin/dashboard`
- 客户管理：`/admin/customers`
- 留言管理：`/admin/messages`
- 产品管理：`/admin/products`
- 订单管理：`/admin/orders`

#### 3. 客服人员（staff）

**权限范围**：
- ✅ 客户查看（只能查看，不能删除）
- ✅ 留言查看与回复（不能删除）
- ✅ 订单查看（不能修改）
- ❌ 产品管理（不能修改产品信息）
- ❌ 用户管理（不能访问）

**访问路径**：
- 登录页面：`/admin/login`
- 仪表盘：`/admin/dashboard`
- 客户管理：`/admin/customers`（只读）
- 留言管理：`/admin/messages`（可回复）
- 订单管理：`/admin/orders`（只读）

#### 4. 销售人员（sales）

**权限范围**：
- ✅ 客户查看（只能查看自己负责的客户）
- ✅ 订单查看（只能查看自己的订单）
- ❌ 留言管理（不能访问）
- ❌ 产品管理（不能访问）
- ❌ 用户管理（不能访问）

**访问路径**：
- 登录页面：`/admin/login`
- 仪表盘：`/admin/dashboard`（受限）
- 客户管理：`/admin/customers`（仅自己的客户）
- 订单管理：`/admin/orders`（仅自己的订单）

## 🔒 安全与权限说明

### 登录安全

**密码要求**：
- 长度至少8位
- 包含大小写字母和数字
- 包含特殊字符（可选）
- 定期修改（建议90天）

**登录限制**：
- 3次登录失败后锁定账号15分钟
- 登录成功后记录登录日志
- 支持登录地点和设备记录
- 支持异地登录提醒

**会话管理**：
- Token有效期：7天
- 支持自动续期
- 支持强制下线
- 支持多设备登录管理

### 权限控制

**基于角色的访问控制（RBAC）**：
- 超级管理员：所有权限
- 管理员：大部分权限
- 客服人员：受限权限
- 销售人员：受限权限

**操作日志**：
- 记录所有重要操作
- 包括操作人、时间、内容、IP地址
- 日志保留时间：1年

**数据加密**：
- 密码使用bcrypt加密存储
- 敏感数据传输使用HTTPS
- 数据库连接使用SSL
